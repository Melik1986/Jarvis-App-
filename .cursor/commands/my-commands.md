# Быстрые команды (сабагенты и задачи)

Используй этот список при вызове `/my-commands` или когда пользователь явно указывает команду.

## Формат вызова

Пользователь пишет после имени команды аргумент (или выбирает номер). Выполни **одну** задачу по выбранной команде.

---

## 1. Поиск в интернете — `/search` или `1`

**Сабагент:** web-search-subagent.

- Действие: выполнить поиск в интернете по запросу пользователя.
- Инструмент: WebSearch. Не выдумывать URL и факты.
- Пример: `/my-commands search: последняя версия React` или `/my-commands 1 React 19`.

---

## 2. MCP / сторонние сервисы и библиотеки — `/mcp` или `2`

**Сабагент:** mcp-tools-subagent.

- Действие: выполнить задачу через MCP (браузер, Supabase, n8n, Figma, Railway, Vercel, Context7, Linear, Playwright и т.д.).
- Порядок: прочитать схему в `mcps/<server>/tools/<tool>.json` → вызвать `call_mcp_tool(server, toolName, arguments)`.
- Пример: `/my-commands mcp: получить док по Zustand из Context7` или `/my-commands 2 Supabase список таблиц`.

---

## 3. Документация по библиотеке — `/docs` или `3`

- Действие: актуальная документация и примеры по библиотеке/сервису.
- Предпочтительно: MCP Context7; при необходимости Web Search.
- Пример: `/my-commands docs: expo-av` или `/my-commands 3 Vercel AI SDK`.

---

## 4. Контекст проекта JSRVIS — `/context` или `4`

- Действие: кратко напомнить цель проекта, модули (Voice, Vision, RAG, интегратор 1С), стек, AI Adapter Pattern.
- Правило: project-context.mdc.

---

## 5. Рефакторинг / линт — `/fix` или `5`

- Действие: исправить код в выделенном файле/фрагменте — линт, типы, стиль по правилам проекта (no any, BEM, DRY, до 20 строк на функцию).
- Не ломать рабочий код; минимальные изменения.

---

## 6. Объяснить код — `/explain` или `6`

- Действие: кратко объяснить выделенный код (что делает, зачем, возможные подводные камни).
- Без лишних абстракций; по делу.

---

## Как выполнить

1. Если пользователь вызвал `/my-commands` без аргумента — ответь списком этих команд и попроси выбрать номер или написать команду с аргументом.
2. Если указан номер (1–6) или имя команды (search, mcp, docs, context, fix, explain) — выполни **только** эту команду для переданного аргумента/контекста.
